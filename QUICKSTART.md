# Flash Token System - Quick Start Guide

## 5-Minute Setup

### 1. Install Dependencies

```bash
npm install
cd frontend && npm install && cd ..
```

### 2. Start Local Network

```bash
npm run dev:contracts
```

Keep this running in one terminal.

### 3. Deploy Contract (New Terminal)

```bash
npm run deploy:ethereum
```

Note the contract address (starts with 0x...).

### 4. Setup Frontend Environment

Edit `frontend/.env`:

```
VITE_FLASH_TOKEN_ADDRESS=0x[your-contract-address]
```

### 5. Start Frontend

```bash
npm run dev:frontend
```

Opens at `http://localhost:3000`

## ✅ Testing the System

1. **Open Frontend**
   - Browser opens automatically to localhost:3000

2. **Connect Wallet**
   - Click "Connect Wallet"
   - Approve MetaMask connection
   - Select Hardhat Local Network (Chain ID 1337)

3. **Get Test Tokens**
   - Balance should show 1,000,000 FLASH (contract owner)
   - If not, check contract address is correct

4. **Try to Send Tokens**
   - Enter test recipient: `0xf39Fd6e51aad88F6F4ce6aB8827279cffFb92266`
   - Enter amount: `100`
   - Click "Send Tokens"
   - Should see error: "Cannot transfer flash tokens without paying gas fee"

5. **Pay Gas Fee**
   - In "Gas Fee Payment" card, enter `0.01` ETH
   - Click "Pay Gas Fee"
   - Confirm in MetaMask
   - Wait for confirmation

6. **Send Tokens Again**
   - Same recipient and amount
   - Should succeed this time
   - Balance updates automatically

## 🛠️ Troubleshooting

### MetaMask says "Network not in menu"

**Solution**: Add Hardhat network to MetaMask
1. Open MetaMask
2. Click Network dropdown
3. Click "Add Network"
4. Fill in:
   - Network: Hardhat Local
   - RPC: http://127.0.0.1:8545
   - Chain ID: 1337

### "Contract code not found at address"

**Solution**: Check deployment
```bash
# Make sure contract deployed successfully
npm run deploy:ethereum
# Copy address and update frontend/.env
```

### "No tokens showing in wallet"

**Solution**: Import token to MetaMask
1. Click "Import tokens"
2. Paste contract address
3. Click "Import"

### Transfer still blocked after gas payment

**Solution**: Refresh page and wait for confirmation
```bash
# Refresh browser
# Wait 10 seconds
# Try transfer again
```
